Plan: Trace GUI → Segmentation Pipeline
TL;DR — Entry point after user scribbles is src/canvas_widget.py::mouseReleaseEvent (emits scribbles_changed); segmentation is started by src/main_window.py::_run_segmentation, which drives MultiClassSegmentationSession and/or EnsembleSegmentationSession (calls seed refinement in mgc_api.mgc_refine_seeds, color-space conversion in color_space.convert_color_space, GrabCut in opencv_grabcut_refine, per-class assembly in segment_all_classes, ensemble majority voting in majority_vote_indexed), and final display is src/canvas_widget.py::set_segmentation_mask (painted in paintEvent). ✅

Steps — exact flow (entry → segmentation → display)
User finishes drawing
src/canvas_widget.py::mouseReleaseEvent — builds ScribbleLayer, appends to self.scribbles, emits scribbles_changed.
UI reacts / user triggers segmentation
src/main_window.py::_run_segmentation — reads canvas.get_annotation_map(), updates session/ensemble_session, spawns SegmentationWorker.
Worker performs per-class segmentation
SegmentationWorker.run() → for each class call:
MultiClassSegmentationSession.segment_class
Build seeds (fg/bg) from annotations
Optional seed refinement: mgc_api.mgc_refine_seeds
Color-space features already prepared via convert_color_space
Run GrabCut: opencv_grabcut_refine (uses OpenCV)
Optional post-smooth: mgc_api.mgc_post_smooth_mask
Per-class results → multi-class mask
MultiClassSegmentationSession.segment_all_classes → resolve overlaps (_combine_masks)
Ensemble (if selected)
EnsembleSegmentationSession runs multiple color-spaces → majority_vote_indexed for final label selection
Return & display
Worker emits final mask → main_window.py::_on_segmentation_complete → canvas.set_segmentation_mask
CanvasWidget.paintEvent draws overlay via _draw_segmentation_overlay
Key files & symbols:

UI / canvas: canvas_widget.py — mouseReleaseEvent, get_annotation_map, set_segmentation_mask, paintEvent
Orchestration: main_window.py — _run_segmentation, SegmentationWorker, _on_segmentation_complete
Core API: grabcut_refine_api.py — MultiClassSegmentationSession, EnsembleSegmentationSession, opencv_grabcut_refine, majority_vote_indexed
Color spaces: src/color_space.py::convert_color_space
Seed-refine / post-smooth: mgc_api.py::mgc_refine_seeds, mgc_post_smooth_mask (uses mgc_core)
Native processing: OpenCV cv.grabCut (used inside opencv_grabcut_refine)
